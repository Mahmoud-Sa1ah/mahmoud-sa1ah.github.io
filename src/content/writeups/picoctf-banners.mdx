# picoCTF Write‚Äëup ‚Äî don‚Äôt you love banners

How a Leaked SSH Banner Led to Root Access. CTF challenges are often built around misconfigurations that look harmless at first glance. In this picoCTF challenge, a simple SSH banner leak was enough to escalate access and retrieve a flag from `/root/flag.txt`.

This write‚Äëup walks you through every step of the exploitation process, from information gathering to abusing symbolic links (symlinks) for privilege escalation.

**Challenge Description:**
Can you abuse the banner? The server has been leaking some crucial information on `tethys.picoctf.net <port>`. Use the leaked information to get to the server. To connect to the running application use `nc tethys.picoctf.net <port>`. From the above information abuse the machine and find the flag in the `/root` directory.
